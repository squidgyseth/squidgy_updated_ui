# N8N Agent Workflow Overview

**Purpose**: High-level overview of how N8N workflows work with Squidgy agents.

**Last Updated**: 2026-03-06

---

## Overview

N8N workflows are automatically generated by the Python agent creation script. Users should NOT manually edit these workflows - all customization is handled through the script's interactive prompts.

## Workflow Generation

When you run `python scripts/create-agent-from-template.py` and choose to generate an N8N workflow:

1. **Template Download** - Downloads standard template `ijDtq0ljM2atxA0E`
2. **Customization** - Replaces placeholders and remaps credentials
3. **Deployment** - Optionally deploys to N8N with your confirmation
4. **Manual Steps** - Only credential verification and publishing in N8N UI

## Post-Generation Requirements

After automatic generation, you only need to:

1. **Open workflow in N8N editor** (using provided URL)
2. **Verify credentials** on flagged nodes
3. **Publish workflow** (click Publish button)
4. **Move to folder** (place in appropriate Squidgy folder)

## What NOT to Do

❌ **Do NOT manually edit workflow nodes**
❌ **Do NOT change webhook paths**
❌ **Do NOT modify credential mappings**
❌ **Do NOT edit respond-to-webhook configuration**

All workflow modifications should be done through the Python script by:
- Editing the agent configuration
- Regenerating the workflow
- Redeploying the updated version
